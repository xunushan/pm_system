# ADR-009: pm-plan 职责收敛（仅沟通与产出）

pm-plan 收窄为仅负责与用户沟通和产出生成，产出后退场，不再承担路由、状态转换或卡片刷新编排。

## Context

pm-plan 原本负责规划对话、输出生成，同时 incidental 承担了路由决策、状态转换和卡片刷新编排。这种职责蔓延使其与路由器和状态管理器的职责重叠。

## Decision

pm-plan 只负责和用户沟通、收集意见、产出目标/专题/阶段/任务规划结构，产出后退场。不处理路由、状态转换或配置。全程聊天沟通：专题构成->目标时间范围->逐专题阶段任务（每专题确认后进下一个）->总览卡片触发写库->给 H5 链接。提问引导策略（含目标时间范围、scheduled_start_date 提问）配置在 pm-plan Skill 里。

## Alternatives Considered

### pm-plan 保留广泛职责（含路由/状态/卡片刷新）
- Rejected: 与路由器和状态管理器职责重叠，归属混乱。

## Consequences

- pm-plan 聚焦且可测试。
- 路由和状态逻辑明确分配给其他组件。
- 影响：05 Skill pm-plan 重写；01 Story 0+1 改写。
