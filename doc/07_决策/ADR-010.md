# ADR-010: pm 主路由纯路由 + 预填 H5 链接

pm 主路由不直接调 Service，仅做路由；配置/调度类指令由 pm LLM 提取参数后生成预填 H5 链接。

## Context

原 pm 主路由既负责分发到正确 Skill，又对配置/调度类指令直接调 Service。这与"pm-plan 不调 Service"的职责不一致（用户质疑"pm-plan 不调 Service，pm 却调"）。

## Decision

pm 主路由不直接调 Service。生成类指令路由到子 Skill（子 Skill 调 Service）；配置/调度类指令由 pm LLM 提取参数->拼成 URL 查询参数->生成预填 H5 链接->页面读取参数预填表单->用户确认提交->页面调 Service。

## Alternatives Considered

### pm 主路由直接调 Service（原方案）
- Rejected: 与 pm-plan 不调 Service 的职责不一致。

## Consequences

- 路由稳定，仅新增路由时才变更。
- 预填链接逻辑集中。
- 请求流程中配置/调度类操作增加一次页面确认步骤。
- 影响：05 Skill pm 主路由改写；06 表 1 配置/调度类路由目标改为"生成预填链接"。
