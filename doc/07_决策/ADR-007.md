# ADR-007: 配置类操作不建 Skill

模板配置、项目空间关联等纯字段 CRUD 不包装为独立 Skill，由配置层直接处理。

## Context

原 v1.2 为每个用户操作建 Skill，包括简单的配置开关，导致 Skill 注册表膨胀，充满无工作流逻辑的一次性操作。

## Decision

模板配置、项目空间关联设置为纯字段 CRUD，走 ConfigAppSvc/WorkspaceAppSvc 代码，不建独立 Skill。对话路径仅用于需 LLM 理解意图的场景（如自然语言描述变更意图），pm 主路由 LLM 提取参数后直接调 Service；看板路径直接 CRUD。

## Alternatives Considered

### 每个配置操作包装为 Skill（原 v1.2）
- Rejected: 纯确定性逻辑（D3 判定标准），统一性不值得维护开销，Skill 注册表膨胀。

## Consequences

- Skill 注册表聚焦工作流逻辑。
- 配置层直接可访问，无间接层。
- 影响：05 Skill 清单不含 pm-config（原 v1.2 有）；04 配置类 API 标注为代码能力。
