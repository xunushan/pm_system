# ADR-016: 前置按今日整体生成 + 与任务解耦

前置任务按"今日整体"生成（不按单个任务），与任务勾选解耦。

## Context

前置原按单个任务生成并绑定。当用户在今日计划卡片上改了任务勾选（取消一个任务），前置要不要跟着变？变动需要 LLM 重新生成前置，但 Service 不调 LLM（架构原则），处理不了。根因：任务和前置若有联动，就需要 LLM 重新生成，与"Service 不调 LLM"冲突。

## Decision

前置按"今日整体"生成，与任务勾选解耦。
- 生成时机：pm-daily 生成今日计划时，调 pm-subtask 生成"今日整体前置"（一次 LLM 调用）。
- 卡片交互：候选任务勾选 + 前置勾选，两组独立，互不影响。取消任务不影响前置。
- 用户只能勾选/取消候选任务（不能加新任务），前置独立勾选可取消。
- 前置不区分任务归属，就是"今日整体的准备"。

## Alternatives Considered

### 按单个任务生成前置（原方案）
- Rejected: 任务和前置联动需要 LLM 重新生成，与"Service 不调 LLM"冲突；且多任务共享准备时冗余。

## Consequences

- 每日仅生成一次前置，内容冗余减少。
- 前置通过上下文引用任务，而非绑定，耦合更松。
- 影响：01/06 Story 3 改写；05 pm-daily 调 pm-subtask 生成今日整体前置。
