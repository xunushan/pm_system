# ADR-015: 去掉"需人工介入"终态

状态机移除"需人工介入"终态，智能体任务重试不通过时保持进行中、由用户手动接管。

## Context

状态机原含一个"需人工介入"终态。该状态在语义上与其他终态重叠（如阻塞任务也需要人工介入），重叠造成"分配哪个状态"的困惑。用户提出"用户介入的话状态是不是不用改变，任务仍然是进行中，完成后用户再去确认任务完成"。

## Decision

移除"需人工介入"终态。智能体任务 3 次重试不通过不改状态，用户手动接管，"/pm 确认完成"后系统重新接管后续智能体任务。需要介入的任务通过当前非终态加标志或注解表达，而非专用终态。

> 进程生命周期：本决策原基于早期多进程设想（"3 次不通过->系统进程退出->不同端口重启"），已被 ADR-020 修正为全局单进程 + 多 session 模型。核心决策（去掉该终态、用户确认后系统重新接管）不变。

## Alternatives Considered

### 保留"需人工介入"终态（原方案）
- Rejected: 状态在语义上与其他终态重叠，分配哪个状态的困惑无法靠文档消除。

## Consequences

- 状态机终态更少，更易推理。
- 介入需求正交追踪（如标志或标签）。
- 已处于被移除状态的任务需迁移。
- 影响：02 tasks 状态去掉"需人工介入"；01/06 Story 4A 改写；新增"/pm 确认完成"指令。
