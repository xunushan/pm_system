# ADR-011: 专题无序 + 阶段 roadmap 强约束

专题去 sort_order（并列无序），阶段作为 roadmap 保持强排序。

## Context

专题（主题分组）和阶段（roadmap 里程碑）原本都强制排序。实践中专题经常被用户重排，使强制排序形同虚设。用户提出"专题无序，只有阶段是有序的，阶段更像 roadmap"。

## Decision

专题去 sort_order，并列无序。阶段是 roadmap，强约束按 sort_order 顺序激活，不可跳。选了专题自动锁定第 1 个未开始阶段。专题无序，故专题完成时给用户专题清单选下一专题（不能自动锁定）。

## Alternatives Considered

### 专题和阶段都强制排序（原方案）
- Rejected: 专题排序约束在实践中被用户忽视，只增加摩擦，无实际价值。

## Consequences

- 专题可自由重排而不违反约束。
- 阶段进展严格校验，乱序阶段转换被拒绝。
- UI 必须区分无序专题和有序阶段。
- 影响：02 themes 去 sort_order；01/06 Story 2/9 衔接逻辑改写。
