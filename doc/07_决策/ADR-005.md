# ADR-005: 状态机扩充——已暂停态 + 有限回退

状态机新增"已暂停"态并支持有限回退，以支持轻量编辑与纠错。

## Context

原状态机单向流转不可回退（02 数据模型 v1.4 第 2.13 节），既无法暂停工作（不完成也不取消），也无法撤销误操作的状态转换。起因是支持轻量编辑/纠错（问题 5）。

## Decision

阶段/任务/专题/目标新增"已暂停"态；支持有限回退（已完成→进行中/待执行）。
- 流转规则：forward（正常）、pause（进行中→已暂停）、resume（已暂停→进行中）、revert（已完成→进行中/待执行，reason 必填）。
- 回退规则：revert 必填 reason；触发即时重算级联（可能把已完成的上级拉回进行中）；所有变更写 status_change_log。
- 已暂停语义：不纳入每日计划推荐、不推 deadline 提醒、不占全局并发名额。

## Alternatives Considered

### 单向流转不可回退（原方案）
- Rejected: 无法暂停工作也无法纠错，不满足轻量编辑/纠错需求（问题 5）。

## Consequences

- 用户可暂停和恢复而不丢失上下文。
- 有限回退保留线性历史，允许一次例外。
- 状态转换校验必须考虑新增的暂停和回退路径。
- 连带调整：放宽 CHECK 约束（原单向流转）；主表不冗余审计字段，新增 status_change_log 表。
- 影响：02 新增 status_change_log 表、放宽 CHECK；03 第 8.8 节状态机约束；04 board/status 接口、错误码 1005。
