# ADR-014: 前置/后置只服务人执行任务

前置/后置子任务只对人执行任务生成，智能体任务不生成。

## Context

前置/后置子任务原对所有任务类型可用。但前置/后置设计用于帮助人类准备和反思，应用到自动任务无价值且增加噪音。用户提出"只有人负责的任务才会有前置和后置子任务，帮人完成准备和收尾工作"。

## Decision

前置/后置子任务仅针对人执行任务生成和展示，自动任务无前置/后置任务。前置/后置的执行主体为智能体（opencode run 自动做归档、生成自测题等）。

## Alternatives Considered

### 所有任务类型都有前置/后置任务（原方案）
- Rejected: 自动任务的前置/后置无意义，浪费生成且增加 UI 噪音。

## Consequences

- 前置/后置任务生成以任务执行类型为条件。
- 自动任务生命周期更简单。
- 影响：01/06 Story 3（前置只对人）、Story 4A（不生成后置）、Story 4B（后置生成）改写。
