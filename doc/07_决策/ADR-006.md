# ADR-006: 项目空间托管模式（managed 字段）

workspaces 新增 managed 字段，区分系统托管与用户关联已有路径两种模式。

## Context

用户新需求："希望能够设置专题关联的项目空间，设置好项目空间的专题不需要进行初始化。" 即有些专题由系统自动初始化（mkdir + git init + 骨架），有些关联用户已有路径、跳过初始化。缺少明确标志时，系统需从间接信号推断模式，不可靠。

## Decision

workspaces 新增 managed 布尔字段：
- managed=1（默认）系统托管，激活时初始化（mkdir + git init + 骨架）。
- managed=0 关联已有路径，激活时跳过初始化；Service 校验 path 存在性，不碰目录结构/git，直接置"已就绪"，系统仍可往该路径读写文件。
- 设置入口属配置类 CRUD，不建 Skill--看板直接编辑（填 path + managed=0）或对话经 pm 主路由 LLM 提取参数后调 Service。
- 主 Agent 进程不受影响：opencode serve 基于 workspace.path 启动，托管/关联一致。

## Consequences

- 显式模式，无需推断。
- 所有调度和状态逻辑检查 managed 标志。
- 新增 API：PUT /api/v1/workspaces/{id}/link。
- 影响：02 workspaces 表新增 managed；03 第 8.11 节工作空间初始化幂等性；04 workspaces/link 接口；01 新增 Story 8。
