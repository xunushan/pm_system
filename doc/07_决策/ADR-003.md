# ADR-003: 今日计划由 LLM 决策

今日计划生成从固定规则改为 LLM 综合决策，以适应上下文因素。

## Context

今日计划原基于固定规则（优先级、截止日期、工作量），无法适应上下文因素，如用户精力、近期阻塞事项、或用自然语言表达的变化优先级。用户指出"当天的任务计划生成需要 LLM 基于昨天完成的和当前阶段下任务综合判断今天的安排"。

## Decision

今日计划由 LLM 综合决策（昨日完成 + 当前阶段进度 + deadline 紧急度 + 任务依赖 + 精力），非纯 deadline 排序。分工：Service 代码负责任务池筛选（捞取待执行、过滤 scheduled_start_date、过滤已暂停）；LLM Skill 负责综合决策今天做什么、做多少、怎么排。

## Alternatives Considered

### 固定规则引擎（原方案）
- Rejected: 无法适应 nuanced 上下文，优先级逻辑僵化，无法处理规则无法表达的上下文规划。

## Consequences

- 计划可适应上下文和自然语言输入。
- 新增 API：GET /api/v1/daily/plans/pool（任务池预查询，供 LLM 输入）。
- 影响：05 新增 pm-daily Skill；04 新增 plans/pool 接口；01 Story 3 主流程改写。
