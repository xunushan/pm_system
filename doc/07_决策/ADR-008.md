# ADR-008: 状态看板载体--自研 H5 页面

规划确认后给 H5 页面链接，所有结构调整在 H5 页面做，替代原先以飞书多维表格作为状态看板的方案。

## Context

第一轮决策曾选飞书多维表格作为状态看板载体（承载专题/阶段/任务的 CRUD、状态可视化、轻量编辑）。调研发现：飞书卡片不支持级联选择（cascader），多维表格 record 变更事件订阅能力（字段级回调粒度、延迟、webhook 支持）待验证，载体绑定多维表格限制交互。现有架构是"飞书 Bot + 卡片回调"，引入"页面"是架构级变更。

## Decision

采用自研 H5 页面作为状态看板与调整入口：规划确认后给 H5 页面链接，所有结构调整（CRUD、deadline、状态变更）在 H5 页面做。页面支持 URL 参数预填（pm 生成带参链接，页面读取参数预填表单）。个人使用，无需鉴权。

## Alternatives Considered

### 飞书多维表格（原第一轮决策，已推翻）
- Rejected: 调研发现飞书卡片不支持级联选择，多维表格 record 变更事件订阅能力待验证，载体绑定多维表格限制交互。

## Consequences

- 更丰富的交互（按钮、表单、条件渲染、URL 参数预填）。
- H5 页面需开发、托管和维护。
- 用户从表格视图切换到表单式界面。
- 影响：01/06 全面改写；03 架构入口 C 从多维表格回调改为 H5 页面调 Service API；02 废弃 BoardAppSvc 反向同步（无多维表格）。
