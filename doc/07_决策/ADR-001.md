# ADR-001: 状态变更即时级联

任务/阶段状态变更时即时触发级联检查，替代原先仅在日终总结时执行的批量级联，以消除状态滞后。

## Context

原方案的级联触发时机绑死日终总结（02 数据模型 v1.4 第 2.14 节："仅在确认日终总结时执行级联检查"），导致状态最长滞后 24 小时，是"级联复杂且滞后"问题（问题 3）的根源。用户要求状态实时可见，不能等待批量作业。

## Decision

级联触发时机从"仅在确认日终总结时"改为"任务/阶段状态变更时即时触发"，级联检查在状态变更的同一事务内执行。

## Alternatives Considered

### 日终批量级联（原方案）
- Rejected: 绑死日终总结导致状态滞后，违反实时可见性需求，是问题 3 的根源。

## Consequences

- 状态始终最新，无滞后窗口。
- 事务边界：级联检查为纯 DB 查询 + UPDATE，与状态变更同事务（<200ms，满足飞书 3 秒回调）；副作用（工作空间初始化、后置子任务、衔接推送）在事务提交后异步执行。
- 连带调整：日终总结退化为纯回顾 + 快照（不再级联）；daily_records.is_confirmed 语义从"级联触发开关"变为"回顾确认标记"。
- 影响：02 第 2.15 节改写；04 tasks/complete、output/confirm、board/status 均含即时级联；daily/summary/confirm 移除级联。
