# Feature: 计划调度与执行 (Scheduling & Execution)

## Glossary
- **衔接（Handoff）** - 一个阶段完成后，系统推送下一阶段激活卡片让用户确认是否开始。用户可感知：收到"XX 已完成，是否开始 YY？"卡片。与"即时更新上级状态"的区别：后者是进度统计刷新，衔接是流程推进到下一阶段。
- **即时（Immediate）** - 用户操作后，页面/聊天立即展示更新结果，无需等待或刷新。

## User Stories

### Story 2: 首次纳入调度（含项目空间设置）

**User Story**  
作为用户，我希望激活规划好的目标中的步骤，开始实际推进；通过卡片完成：选专题 -> 填 deadline -> 确认激活。项目空间默认托管初始化，需要关联已有路径或关闭初始化时走配置页。

**When**  
用户发送"开始调度"或 H5 页面点"激活"。

**Dependencies**  
目标已存在且状态为"未开始"或"进行中"。

**Workflow**  
```
用户：/pm 开始调度

PM助手：（卡片 A：选专题）
    多选要推进的专题（专题无序，并列）：
    ☑ 深度学习基础
    ☐ 面试准备
    ☐ 具身智能专项
    ☐ Agent 框架
    [下一步]
    默认全托管初始化项目空间，调整 managed/关联路径请前往配置页

用户：勾选"深度学习基础"和"面试准备"，点"下一步"

PM助手：（卡片切换为 B：填 deadline）
    将激活：
    · 神经网络基础与 MLP（深度学习基础）   deadline：[选择日期]
    · 算法基础（面试准备） deadline：[选择日期]
    [确认调度]

用户：填各 deadline，点"确认调度"

PM助手：调度已确认，2 个阶段已激活。
    · 神经网络基础与 MLP：项目空间初始化中（托管）
    · 算法基础：项目空间初始化中（托管）
    从今天起纳入每日计划。
```

**Acceptance Criteria**  
- 采用两步卡片：第 1 步选专题，确认后进入第 2 步填 deadline
- 支持多选专题，每个专题自动激活其第 1 个未开始阶段
- 阶段自动锁定（强约束，用户不选择具体阶段）
- 项目空间默认托管初始化（managed=true：系统生成路径 + 异步 mkdir/骨架）；卡片不设 managed/path 入口
- 需要关联已有路径（managed=false）或关闭初始化时，走 H5 配置页（激活前设置）
- deadline 必填
- 全局进行中阶段数 ≤ 3（已暂停不占名额）
- 激活时同步将专题/目标状态置为进行中
- 记录阶段激活时间，每日计划以该时间为准
- 首次激活与中途追加走同一流程
- 激活后不能修改项目空间模式

**Boundary Scenarios**  
- 用户选 0 个专题点"下一步" -> 系统提示至少选择 1 个，阻止进入下一步
- 配置页关联已有路径但路径不存在 -> 配置页提示并阻止保存（卡片 A 不收 path，不在此提示）
- 所选专题激活后全局进行中将超过 3 -> 系统阻止本次激活并提示先完成或暂停现有阶段（已暂停不占名额）

**Key Decisions**  
> - **分两步收集信息** - 先选专题，再填 deadline，避免单张卡片承载过多字段。
> - **多选专题** - 跨专题并行推进是实际需求，一次激活多个专题。
> - **项目空间默认托管初始化** - 卡片只选专题；managed/path 不在卡片设置（默认托管，调整走配置页），降低卡片复杂度。
> - **阶段强约束** - 专题内阶段按顺序激活，自动锁定第 1 个未开始阶段。
> - **计划开始日期在 Story 1 确认** - 本 Story 不修改，激活后以阶段激活时间推送。

### Story 3: 当日计划推送

**User Story**  
作为用户，我希望每天收到 PM 助手推送的当日任务清单，由系统综合昨日完成、进度、deadline、依赖生成，卡片带候选任务供勾选。

**When**  
- 定时：8:30 自动
- 主动：用户发送"今日计划"
- 巡检：10:00 未确认计划时提醒

**Dependencies**  
存在已激活阶段（已记录阶段激活时间）。

**Workflow**  
```
（8:30 定时触发）

PM助手：📋 今日计划（2026-07-02）
    昨日完成：反向传播推导 ✅、手写 MLP PyTorch ✅

    今日候选任务（勾选今天要做的，3-5 个）：
    ☑ 优化器专题（神经网络基础与 MLP(7/15)）[人]
    ☑ 手写 MLP PyTorch（神经网络基础与 MLP(7/15)）[人]
    ☐ MNIST 端到端训练（神经网络基础与 MLP(7/15)）[人]
    ☐ 数组专题 2 题（算法基础(7/20)）[人]
    ☐ 项目复盘提纲（面试准备(7/25)）[智能体]

    执行前工作（勾选要执行的，可取消）：
    ☑ 搜集优化器相关资料
    ☑ 准备 PyTorch 开发环境
    ☐ 获取今日 LeetCode 题目
    [确认今日计划]

用户：勾选 3 个任务 + 取消"获取今日 LeetCode 题目"，点"确认今日计划"

PM助手：今日计划已生效，执行前工作进行中。

（5 分钟后）
PM助手：执行前工作已完成 ✅
    · 优化器资料已搜集
    · PyTorch 环境已就绪
    你可以开始学习了。
```

**Acceptance Criteria**  
- 候选任务由系统综合昨日完成情况、进度、deadline、依赖生成，数量为 5–8 个供勾选
- 执行者（人/智能体）根据专题类型推断，卡片只读展示（修改走 H5 页面）
- 候选任务展示格式：任务名（阶段名(deadline)）[执行者]
- 候选任务仅来自已激活阶段，已暂停阶段不出现
- 执行前工作按"今日整体"生成，与任务解耦，两组独立勾选
- 执行前工作仅对执行者=人的任务生成，可勾选/取消
- 用户只能勾选/取消候选任务，不能新增任务
- 用户勾选 3–5 个任务后方可确认当日计划
- 确认当日计划后立即生效
- 10:00 前未确认，系统提醒 1 次，不自动执行
- 任务完成走 Story 4B，今日计划卡片不承担完成确认

**Boundary Scenarios**  
- 用户勾选任务少于 3 个 -> 系统阻止确认并提示需勾选 3–5 个
- 当日候选任务全部为智能体执行 -> 执行前工作区域不展示（前置仅服务人执行任务）

**Key Decisions**  
> - **执行前工作按今日整体生成并与任务解耦** - 不按单个任务生成，两组独立勾选，取消任务不影响前置。
> - **执行前工作只服务于人执行任务** - 前置是帮人做执行准备，智能体任务无需准备。

### Story 4A: 工作空间任务执行 - 智能体（代码开发、文档生成）

**User Story**  
作为用户，我希望由智能体执行代码开发/文档生成类任务，完成后对产出文件进行验收。

**When**  
当日计划确认后，自动触发（执行者=智能体的任务）。

**Dependencies**  
已确认当日计划，且存在执行者=智能体的勾选任务。

**Workflow**  
```
系统：判断勾选任务是否有执行者=智能体？
  - 是 -> 查工作空间路径，启动执行环境（首次），下发任务
  - 设置单次执行超时 2h

执行环境：执行，按项目空间规范（claude.md 类）写产出文件

（产出完成）

系统：记录产出 -> 发验收卡片 + 发送产出文件到飞书：
    📋 任务完成确认
    任务：设计消息 Schema
    产出文件：
    · 消息Schema-v1.md
    · schema-test.py
    （按钮：验收通过 / 需要修改）
    + 文件消息（用户在飞书直接看）

用户：看文件，点"验收通过"

系统：标记任务已完成 + 即时更新上级状态 -> 触发阶段衔接

（若 3 次验收不通过）
系统：停止自动执行，验收卡片刷新为"建议手动接管"提示（含工作空间路径 + 会话 ID，无额外飞书通知）

用户：用卡片给出的会话 ID 在本地接管该任务，给详细指令驱动产出，把关

用户：/pm 确认完成 设计消息 Schema

系统：标记完成 + 即时更新上级状态 + 重新接管后续智能体任务
```

**Acceptance Criteria**  
- 智能体任务在当日计划确认后自动开始执行，不阻塞用户其他操作
- 产出统一为文件，按项目空间规范写入
- 推送验收卡片，并将产出文件发送到飞书（用文件名体现产出）
- 验收通过后同步更新任务状态为完成并触发阶段衔接
- "需要修改"通过卡片输入框直接提交，重试上限 3 次
- 连续 3 次验收不通过：停止自动执行，验收卡片刷新为"建议手动接管"提示（含工作空间路径 + 会话 ID），不发额外飞书通知
- 用户手动处理后，通过"/pm 确认完成"指令标记完成；系统随后重新接管后续智能体任务
- 单次执行超过 2 小时未完成，触发超时告警

**Boundary Scenarios**  
- 待定义：用户点"需要修改"但未输入反馈内容时，系统是否允许提交（原文未约定）
- 待定义：执行超时（2 小时）时通知用户的具体文案（原文未约定）
- 待定义：用户发送"/pm 确认完成 [任务名]"时任务名不存在或已完成，系统如何反馈（原文未约定）

**Key Decisions**  
> - **即时更新上级状态** - 任务完成即时向上更新状态，不再依赖日终总结。
> - **智能体任务不生成执行前/后置** - 前置/后置只服务人执行任务，帮人完成准备和收尾。
> - **不设"需人工介入"终态** - 3 次不通过不改任务状态，用户介入后用"/pm 确认完成"，系统重新接管后续智能体任务。

### Story 4B: 工作空间任务执行 - 人（学习、刷题）

**User Story**  
作为用户，我希望完成学习/刷题类任务后通过指令标记完成，并可选择性生成后置收尾工作。

**When**  
用户学习/任务完成后，主动发送"/pm 完成 [任务名]"。

**Dependencies**  
任务处于进行中状态。

**Workflow**  
```
用户：/pm 完成 反向传播推导

PM助手：（子任务生成环节执行）
    已标记完成：反向传播推导
    即时更新上级状态：MLP 阶段 3/6，仍进行中。

    后置工作（勾选要执行的，可全取消）：
    ☑ 1. 笔记归档
    ☑ 2. 总结生成
    ☑ 3. 自测题生成
    [确认后置] [不需要后置]

用户：取消勾选"总结生成"，点"确认后置"

PM助手：后置工作已启动，完成后通知你。

（10 分钟后）
PM助手：后置工作已完成 ✅
    · 笔记已归档
    · 自测题已生成
```

**Acceptance Criteria**  
- 人完成任务后通过"/pm 完成 [任务名]"指令触发
- 标记完成时即时向上更新状态，并生成后置工作清单
- 任务在标记完成时即视为完成，不等后置确认（脱钩）
- 后置清单仅对人执行的任务生成
- 后置子任务确认后立即异步执行
- 后置清单以卡片勾选，默认全选，用户可取消不想要的项，不能新增，可全部取消
- 后置任务异步执行，失败仅通知、不阻塞主流程

**Boundary Scenarios**  
- 待定义："/pm 完成 [任务名]"匹配到多个同名任务时，系统如何让用户选择（原文未约定）
- 待定义：用户发送"/pm 完成"但不带任务名时，系统如何反馈（原文未约定）

**Key Decisions**  
> - **后置和完成脱钩** - 完成时先标记（即时更新上级状态），后置是可选收尾，用户可全部取消。
> - **两个完成指令** - 4B 用"/pm 完成"（带后置生成），4A 人工确认用"/pm 确认完成"（不带后置）。

### Story 5: 日终总结

**User Story**  
作为用户，我希望每天结束时查看当日任务完成情况，对状态有异议的任务通过勾选确认批量修正（并即时更新上级状态），确认后归档快照。

**When**  
- 主动：用户发送"今日总结"
- 定时：21:00 提醒

**Dependencies**  
当日存在计划记录。

**Workflow**  
```
用户：/pm 今日总结

PM助手：📊 今日总结（2026-07-02）
    今日任务（勾选=已完成，调整后点确认）：
    ☑ 优化器专题
    ☑ 手写 MLP PyTorch
    ☐ 数组专题 2 题
    阶段进展：MLP 5/6 进行中
    建议：MLP 进度过半，保持节奏。
    [确认日终总结]

用户：数组题其实完成了，勾上"数组专题 2 题"

用户：点"确认日终总结"

PM助手：（对比初始状态反转变化任务 + 更新状态 + 即时更新上级状态 + 重新统计 + 刷新卡片）
    今日任务最终状态：
    · 优化器专题 ✅
    · 手写 MLP PyTorch ✅
    · 数组专题 2 题 ✅
    阶段进展：MLP 6/6 已完成
    日终总结已确认，当日快照已更新。
```

**Acceptance Criteria**  
- 任务状态以勾选框呈现（勾选=已完成），已完成任务默认勾选
- 用户调整勾选后点"确认日终总结"批量提交，不立即生效
- 确认提交时对比初始状态反转变化的任务，更新任务状态并触发即时更新上级状态
- 确认后刷新卡片为最终状态 + 阶段进展统计
- 异议通过勾选框调整（不依赖聊天语义识别），状态双向（勾=完成，取消=未完成）
- 确认后写入当日快照并标记已确认
- 不确认不强制，不影响底层数据

**Boundary Scenarios**  
- 用户误勾 -> 可再次取消勾选，确认前可任意调整
- 待定义：当天无计划记录时发送"/pm 今日总结"，系统如何反馈（原文仅约定前置"当日存在计划记录"，未明示无记录时的反馈）
- 待定义：日终总结未确认时数据保留多久、次日是否还能确认昨日（原文未约定）

**Key Decisions**  
> - **异议走勾选框 + 确认批量提交** - 任务以勾选框呈现（勾=完成），调整后点确认一次性提交，避免逐条立即生效的误操作，也避免聊天语义识别歧义。

### Story 6: 周总结

**User Story**  
作为用户，我希望每周日收到本周回顾报告，确认后归档。

**When**  
周日 12:00 定时触发。

**Dependencies**  
当周存在计划记录。

**Workflow**  
```
（周日 12:00）

PM助手：📊 本周总结（2026-W27）
    每日回顾、阶段健康度、产出统计
    下周建议（参考阶段衔接状态）：
    · 算法基础剩余 1 个任务，建议周一突破
    · CNN 周二激活（与阶段衔接一致）
    [已阅]

用户：点"已阅"

PM助手：周报快照已更新。
```

**Acceptance Criteria**  
- 纯回顾，只读统计，不修改任何状态
- 下周建议参考阶段衔接状态
- "已阅"仅归档并写入周报快照
- 不点"已阅"不强制
- 数据有误需修正：前往 H5 页面（周总结不提供改状态入口）

**Key Decisions**  
> - **周总结不改状态** - 跨度长、隔周改影响大；数据有误只去 H5 页面修正，周总结纯只读回顾。

### Story 8: 主动巡检与阶段衔接

**User Story**  
作为用户，我希望 PM 助手主动掌握进度，阶段完成后主动衔接下一阶段，并在 deadline 临近、未做计划/总结时主动提醒。

**When**  
- 事件即时：阶段/专题完成
- 定时巡检：计划开始日期到了未激活、deadline 临近、未做计划/总结、阶段衔接 24h 未响应

**Dependencies**  
目标已激活且未暂停。

**Workflow**  
```
（场景1：阶段完成 -> 即时衔接）
PM助手：✅ 阶段【神经网络基础与 MLP】已完成
    下一阶段：卷积神经网络（自动锁定，强约束）
    建议 deadline：7/25
    deadline：[选择日期]
    [确认激活] [暂不激活]

用户：填 deadline，点"确认激活"

PM助手：已激活【卷积神经网络】，纳入明日计划。

（场景2：专题完成 -> 选下一专题）
PM助手：✅ 专题【深度学习基础】全部完成
    还有 3 个未完成专题，要激活哪个？
    ○ 面试准备
    ○ 具身智能专项
    ○ Agent 框架
    [下一步] [暂不激活]

用户：选"面试准备"，点"下一步"

PM助手：（卡片切换为填 deadline）
    将激活阶段：算法基础（面试准备第1个未开始阶段）
    deadline：[选择日期]
    [确认激活]

（场景3：计划开始日期到了未激活）
PM助手：你计划 7/2 开始推进，今天到了，要激活吗？
    [去激活] [暂不]

（场景4：deadline 临近）
PM助手：⏰ 【算法基础】deadline 7/20（明天），1/3 完成。
    [去页面调整]
```

**Acceptance Criteria**  
- 阶段完成即时推送衔接卡片（含 deadline 填写）
- 专题完成时给出未完成专题清单供单选，确认后进入填 deadline 步骤
- 计划开始日期到达但未激活时提醒激活，每天 1 次，激活后停止
- deadline 临近时提醒并跳转页面修改 deadline
- 未确认当日计划（10:00）/未做日终总结（21:00）定时提醒
- 阶段衔接卡片 24 小时未响应，再推送 1 次
- 已暂停实体不巡检
- 每项提醒每天最多 1 次

**Boundary Scenarios**  
- 待定义：用户点"暂不激活"后，系统多久再提醒、提醒几次后停止（原文仅约定部分提醒每天 1 次，未约定停止条件）
- 待定义：阶段衔接卡片 24 小时未响应再次推送时，内容是否与首次相同（原文未约定）
- 待定义：用户同时收到多个衔接卡片（多个阶段完成）时，卡片是合并还是分别发送（原文未约定）

**Key Decisions**  
> - **阶段衔接带 deadline 填写** - 衔接卡片形态同 Story 2 卡片 B。
> - **专题完成给专题清单选** - 专题无序，不能自动锁定下一专题。

## Non-Goals
- 不实现日终总结自动确认（须用户手动确认）
- 不实现周总结的状态修改（纯只读）
- 不实现自动阶段衔接（须用户确认激活）
- 不实现智能体任务的执行前/后置工作（仅服务人执行任务）
- 本期不实现卡顿监测
