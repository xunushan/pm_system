# Feature: 执行前后模板配置 (Pre/Post Execution Template Config)

## User Stories

### Story 7: 配置执行前后的固定工作

**User Story**  
作为用户，我希望在 H5 页面配置每个阶段/专题执行前后的固定工作，支持直接增删改查。

**When**  
用户在 H5 页面"子任务模板"板块编辑。

**Dependencies**  
用户已完成目标规划（Story 1）。

**Workflow**  
```
用户：（在 H5 页面"子任务模板"板块）
    模板列表：
    · 准备题目与代码框架（前置，算法基础阶段）
    · 笔记归档（后置，深度学习基础专题）
    · 自测题生成（后置，MLP 阶段）
    [+ 新增模板]

用户：点"新增模板"，在表单选所属专题/阶段 + type + 填 name + description -> 保存

系统：已保存，下次生成计划时自动应用。
```

**Acceptance Criteria**  
- H5 页面模板列表直接展示，每条标注所属专题/阶段
- 新增模板时在表单中选择所属专题/阶段、类型、名称、描述
- 配置粒度为专题或阶段；阶段级覆盖专题级，同名时取阶段级模板
- 配置时不校验专题类型（生成计划时自动跳过智能体任务）
- 删除可恢复
- 保存立即生效

**Key Decisions**  
> - **配置入口为 H5 页面** - 以列表直接展示，不提供聊天指令入口。

---

## Non-Goals
- 不提供文件上传/下载

## Notes
- 模板配置属于"静态配置"，与 Story 3（执行前生成）、Story 4B（后置生成）的动态生成形成"配置-执行"闭环
- 模板仅对执行者=人的任务生效（智能体任务在生成时自动跳过）
