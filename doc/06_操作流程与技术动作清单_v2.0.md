# 卡片按钮路由清单（入口 B）

> 飞书卡片按钮点击回调的硬编码路由表。每行一个按钮：文案 / action_id 或 btn_name / 对应 Story / Service 处理。
> 卡片样式（几态/组件/JSON）见 `doc/09_卡片交互样式记录.md`；集成验证事实见 `doc/13_原型验证/`（飞书卡片验证 + opencode 验证）。
> 用户指令（/pm 类）见各 Skill；H5 操作见 doc/04。

## 两类按钮路由（关键，别混淆）

| 类型 | 识别字段 | 路由方式 | 示例 |
|------|---------|---------|------|
| form 外回传按钮 | `event.action.value.action_id` | 按 action_id 路由 | story1_确认方案 |
| form 内提交按钮 | `event.action.name`（无 action_id） | 按 btn name 路由 | btn_pass / confirm_btn |

> form_submit 按钮不能带 behaviors（V1），靠 name 区分；form 外按钮靠 action_id。详见 `doc/13_原型验证/飞书卡片验证.md` V1/V8。

## 按钮 -> 路由对照

| 按钮 | action_id / btn_name | Story | Service 处理 |
|------|----------------------|-------|-------------|
| 确认方案 | story1_确认方案 | S1 | 方案确认（落库 + 给 H5 链接） |
| 下一步（选专题） | next_btn | S2 | patch 卡片为填 deadline |
| 确认调度 | confirm_btn | S2 | 调度确认（+即时级联 + managed 初始化） |
| 确认今日计划 | confirm_btn | S3 | 每日计划确认 |
| 验收通过 | btn_pass | S4A | 产出验收（+即时级联） |
| 需要修改 | btn_reject | S4A | 产出退回（重试 / 通知） |
| 确认后置 | confirm_btn | S4B | 后置确认（INSERT 后置，异步执行） |
| 不需要后置 | btn_no_post | S4B | 后置确认（不插入，任务已完成） |
| 全不选 | btn_unselect_all | S4B | update_card 把所有 checker checked=false |
| 确认日终总结 | confirm_btn | S5 | 日终总结确认（不级联） |
| 已阅周总结 | story6_已阅周总结 | S6 | 周总结确认 |
| 确认激活 | btn_activate | S8 | 调度激活（阶段衔接） |
| 暂不激活 | story8_暂不激活 | S8 | 记录，24h 后再提醒 |
| 去激活（开始日提醒） | story8_去激活 | S8 | 跳 S2 激活流程 |
| 去页面调整（deadline 临近） | story8_去页面调整 | S8 | 跳 H5 页面 |

> 说明：S5 日终总结的任务状态修正走 checker 勾选（勾=完成），不单独走按钮路由，随 confirm_btn 批量提交反转。
